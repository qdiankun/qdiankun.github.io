产生过程，我在本地blog是可以hexo g，并且可以hexo s后本地可以正常打开，现在我要提交oschina中，刚开始时，我想不要把 public/ node_modules/ 两个文件夹提交上去，于是:

最后这达到了效果，两个文件夹是没有提交上去，也没有track，但出现另外一个问题，我从 oschina上 clone 下来后 hexo clean -> hexo g ，后只生成了js css 文件，没有 .md文件 。

最终用来如下笨方法解决问题：

1.首先将本地的blog文件下的所有文件都提交到 oschina上去
2.从oschina山下载你提交上去的blog
3.现在修改source/_post 文件夹下的 .md 文件 重新hexo clean , hexo g 后 发现追踪到了sourc 、pulic 、node_modules下的文件变化，不过我们不想对pulic 、node_modules下的变化提交，要做的是：

首先将public/ .node_modules/ 从本地git种移除

然后 在.gitignore对这两个文件夹忽略

至此，以后再hexo g ，就不会再提示你提交public、node_modules下两个文件夹下的变化
